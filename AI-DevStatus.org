* What Works

- =py-dblock.cs -i updateDblocks <file...>= — expands all registered dblocks in
  listed org files (pure Python, no Emacs needed). Non-file args are skipped;
  unregistered signatures are left untouched.
- =updateDblock.expandAll(filePath)= — single-pass regex dispatch over a file.
- =dblock_particulars= — handles =b:ai:file/particulars=; produces correct absolute
  Working Directory and File fields. Each handler module has a docstring cross-
  referencing its elisp counterpart in =/bisos/blee/env3/dblocks/=.
- =dblock_runResult= — handles =bx:dblock:global:run-result-stdout=; runs =:command=
  via =subprocess.run(shell=True)=, decodes org-mode backslash escapes, appends
  trailing blank line to match elisp output format.
- =py3/tests/verify.sh= — copies =.blank= → =.pydb=, runs =py-dblock.cs=, diffs
  against =.eldb= reference. Remaining diffs are timestamp and filename only.

* Key Design Decisions

- *Import subpackage is =bisos.pyDblock=* — the distribution name auto-derives
  as =bisos.py-dblock= from the repo dir, but a dash is not a valid Python
  identifier, so the importable subpackage is CamelCase =pyDblock=.

- *Engine / handler split* — =updateDblock.py= is signature-agnostic dispatch;
  each dblock signature lives in its own self-registering handler module
  (=dblock_particulars.py=, =dblock_runResult.py=). New signatures are added by
  new handler modules, imported in =py-dblock.cs= for their registration side effect.

- *Exactly 4-hash markers* — the regex uses =#{4}\+= (not =###\#+=) to require
  exactly 4 hashes. Files contain 3-hash markers inside dblock bodies (org-mode
  internal content); matching 3+ would corrupt those.

- *Backslash unescaping in =_parseArgs=* — org-mode stores =\\= in =:command= args
  as a literal double-backslash in the file. =_parseArgs= decodes =\\\\= → =\\= after
  quote-stripping so the shell receives the intended single-backslash sequence.

- *Trailing blank line in run-result-stdout* — elisp =shell-command-to-string=
  appends a trailing newline that org-mode renders as a blank line before =####+END:=.
  The Python handler matches this to keep =.pydb= and =.eldb= diffs clean.

- *=README.org= is the authoritative source* — =py3/README.rst= is generated
  from it by =pypiProc.sh=; never hand-edit the RST.
